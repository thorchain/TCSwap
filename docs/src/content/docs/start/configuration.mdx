USwap uses a centralized configuration system called `USwapConfig` that manages all settings across [toolboxes](/start/toolbox-usage), [plugins](/guides/create-plugin), and integrations. This guide covers how to configure USwap for your application.

## Configuration Overview

USwap's configuration is managed through the `USwapConfig` store, which handles:

- RPC URLs for blockchain connections
- API keys for external services
- Environment settings
- Integration configurations
- Custom API implementations
- Midgard API URLs for THORChain/Maya
- Custom API headers for authentication

## Basic Configuration

When creating a USwap instance, you can provide configuration options:

```typescript
// @noErrorValidation
import { createUSwap } from '@tcswap/sdk';

const uSwap = createUSwap({
  config: {

    apiKeys: {
      walletConnectProjectId: 'your-project-id',
      blockchair: 'your-blockchair-key',
      uSwap: 'your-uswap-key'
    },

    rpcUrls: {
      ETH: 'https://eth-mainnet.g.alchemy.com/v2/your-key',
      AVAX: 'https://api.avax.network/ext/bc/C/rpc'
    },

    midgardUrls: {
      THOR: 'https:
      MAYA: 'https:
    },

    apiHeaders: {
      midgard: {
        'Authorization': 'Bearer your-token',
        'X-Custom-Header': 'custom-value'
      }
    }
  }
});
```

## Midgard API Configuration

Midgard provides historical data and liquidity information for THORChain and Maya. USwap allows you to configure custom Midgard endpoints and authentication headers.

### Custom Midgard URLs

Override the default Midgard endpoints:

```typescript
// @noErrorValidation
import { createUSwap, Chain } from '@tcswap/sdk';

const uSwap = createUSwap({
  config: {
    midgardUrls: {
      [Chain.THORChain]: 'https:
      [Chain.Maya]: 'https:
    }
  }
});





```

### API Headers for Authentication

Add custom headers for Midgard API authentication:

```typescript
// @noErrorValidation
import { USwapConfig } from "@tcswap/sdk";

USwapConfig.setApiHeaders("midgard", {
  "X-Client-ID": "your-client-id",
});

const uSwap = createUSwap({
  config: {
    apiHeaders: {
      midgard: {
        "X-Client-ID": "your-client-id",
      },
      thornode: {
        "X-Client-ID": "your-client-id",
      },
      uSwapApi: {
        "X-API-Key": "additional-api-key",
      },
    },
  },
});
```

### Runtime Updates

Update Midgard configuration at runtime:

```typescript
// @noErrorValidation
import { USwapConfig, Chain } from '@tcswap/sdk';


USwapConfig.setMidgardUrl(Chain.THORChain, 'https:


USwapConfig.setApiHeaders('midgard', {
  'X-Client-ID': 'new-client-id'
});


const midgardUrls = USwapConfig.get('midgardUrls');
const apiHeaders = USwapConfig.get('apiHeaders');
```

## API Headers Configuration

USwap supports custom headers for all API endpoints (Midgard, THORNode, and USwap API). This is useful for:

- Authentication with private API instances
- Custom client identification
- Additional API key management

### Supported APIs

1. **Midgard API** (`midgard`): Historical data and liquidity information
2. **THORNode API** (`thornode`): Direct blockchain node access
3. **USwap API** (`uSwapApi`): Swap quotes, prices, and token data

### Setting Headers

```typescript
// @noErrorValidation
import { USwapConfig } from "@tcswap/sdk";

USwapConfig.set({
  apiHeaders: {
    midgard: {
      "X-Client-ID": "your-client-id",
    },
    thornode: {
      "X-Client-ID": "your-client-id",
    },
    uSwapApi: {
      "X-API-Key": "additional-api-key",
    },
  },
});

USwapConfig.setApiHeaders("midgard", {
  "X-Client-ID": "new-client-id",
});
```

## Complete Example

Here's a production-ready configuration example:

```typescript
// @noErrorValidation
import { createUSwap, USwapConfig, Chain } from '@tcswap/sdk';


const config = {

  apiKeys: {
    walletConnectProjectId: process.env.WALLETCONNECT_PROJECT_ID!,
    blockchair: process.env.BLOCKCHAIR_API_KEY,
    uSwap: process.env.USWAP_API_KEY
  },


  envs: {
    isDev: process.env.NODE_ENV === 'development',
    isStagenet: process.env.USE_TESTNET === 'true'
  },


  rpcUrls: {
    [Chain.Ethereum]: process.env.ETH_RPC_URL || 'https:
    [Chain.Avalanche]: process.env.AVAX_RPC_URL || 'https:
    [Chain.BinanceSmartChain]: process.env.BSC_RPC_URL || 'https:
  },



  midgardUrls: process.env.MIDGARD_URL ? {
    [Chain.THORChain]: process.env.MIDGARD_URL,
    [Chain.Maya]: process.env.MAYA_MIDGARD_URL || process.env.MIDGARD_URL
  } : undefined,


  apiHeaders: {
    midgard: process.env.CLIENT_ID ? {
      'X-Client-ID': process.env.CLIENT_ID
    } : undefined,
    thornode: process.env.CLIENT_ID ? {
      'X-Client-ID': process.env.CLIENT_ID
    } : undefined,
    uSwapApi: process.env.ADDITIONAL_API_KEY ? {
      'X-API-Key': process.env.ADDITIONAL_API_KEY
    } : undefined
  }
};


const uSwap = createUSwap({ config });


if (userPrefersFastNode) {
  USwapConfig.setRpcUrl(Chain.Ethereum, 'https:
}


if (userHasRadixWallet) {
  USwapConfig.setIntegrationConfig('radix', {
    dAppDefinitionAddress: process.env.RADIX_DAPP_ADDRESS!,
    applicationName: 'My USwap App',
    applicationVersion: '1.0.0',
    network: {
      networkId: config.envs.isStagenet ? 2 : 1,
      networkName: config.envs.isStagenet ? 'testnet' : 'mainnet'
    }
  });
}
```

### API Key Validation

Verify your API keys are working:

```typescript
// @noErrorValidation
import { USwapApi } from "@tcswap/sdk";

try {
  const price = await USwapApi.getPrice("ETH");
  console.log("USwap API key is valid");

  console.error("USwap API key issue:", error);
}
```
