## Choose Your Implementation

USwap offers two implementation approaches to suit your needs:

<Tabs>
  <TabItem label="USwap SDK">
    <div>
      <h3>Quick and Simple</h3>
      <p>The all-in-one solution that bundles Core with all available plugins and wallet integrations.</p>
      <p>Perfect for developers who want to get started quickly with minimal configuration.</p>

      **Pros:**
      - Simplicity and convenience
      - Reduced setup time
      - Pre-configured with all available plugins and wallets
      - Faster implementation for multi-chain applications

      **Cons:**
      - Larger bundle size
      - Less flexibility for specific customizations

      <LinkButton href="#sdk-quick-start">Start with SDK</LinkButton>
    </div>

  </TabItem>
  <TabItem label="USwap Core">
    <div>
      <h3>Flexible and Customized</h3>
      <p>The foundation package providing essential functionality.</p>
      <p>Designed to be modular and flexible, allowing you to customize your implementation with only the specific components you need.</p>

      **Pros:**
      - Lightweight bundle size
      - Fine-grained control over dependencies
      - Allows selective updates of components
      - Better for specialized applications

      **Cons:**
      - Requires more configuration
      - More imports and setup code

      <LinkButton href="#core-setup">Start with Core</LinkButton>
    </div>

  </TabItem>
</Tabs>

<Aside type="note">
  Choose the implementation that best suits your project's needs. USwap SDK is
  recommended for most users getting started.
</Aside>

## USwap SDK: Quick Start

The fastest way to integrate USwap into your application.

<Steps>
1. **Install the SDK**

   Choose your package manager:

   <Tabs syncKey="package-manager">
     <TabItem label="bun">```bash bun add @tcswap/sdk ```</TabItem>
     <TabItem label="pnpm">```bash pnpm add @tcswap/sdk ```</TabItem>
     <TabItem label="npm">```bash npm install @tcswap/sdk ```</TabItem>
     <TabItem label="yarn">```bash yarn add @tcswap/sdk ```</TabItem>
   </Tabs>

2. **Initialize USwap**

   ```typescript
   import { createUSwap } from "@tcswap/sdk";

   const defaultUSwap = createUSwap();

   const uSwap = createUSwap({
     config: {
       apiKeys: {
         uSwap: "your-uswap-api-key",
       },
     },
   });
   ```

3. **Start Using USwap**

   Connect a wallet:

   ```typescript
   import { createUSwap, Chain } from "@tcswap/sdk";

   const uSwap = createUSwap({
     config: { apiKeys: { uSwap: "your-uswap-api-key" } },
   });

   const wallet = await uSwap.connectKeystore(
     [Chain.Ethereum],
     "your mnemonic phrase here"
   );

   const address = uSwap.getAddress(Chain.Ethereum);
   ```
</Steps>

## USwap Core: Advanced Setup

For developers who need more control and custom configurations.

<Steps>
1. **Install Core Package**

   Start with the core package:

   <Tabs syncKey="package-manager">
     <TabItem label="bun">```bash bun add @tcswap/core ```</TabItem>
     <TabItem label="pnpm">```bash pnpm add @tcswap/core ```</TabItem>
     <TabItem label="npm">```bash npm install @tcswap/core ```</TabItem>
     <TabItem label="yarn">```bash yarn add @tcswap/core ```</TabItem>
   </Tabs>

2. **Install Required Components**

   Add only the components you need:

   <Tabs>
     <TabItem label="Wallets">
       <Tabs syncKey="package-manager">
         <TabItem label="bun">
           ```bash
           bun add @tcswap/wallets
           ```
         </TabItem>
         <TabItem label="pnpm">
           ```bash
           pnpm add @tcswap/wallets
           ```
         </TabItem>
         <TabItem label="npm">
           ```bash
           npm install @tcswap/wallets
           ```
         </TabItem>
         <TabItem label="yarn">
           ```bash
           yarn add @tcswap/wallets
           ```
         </TabItem>
       </Tabs>

       Install to add integrations with specific wallets.
       Import specific wallet integrations as needed (e.g., `keystoreWallet` from `@tcswap/sdk`).
       Supports around 20+ wallets, including Ledger, Trezor, Coinbase, WalletConnect and more.
     </TabItem>

     <TabItem label="Plugins">
       <Tabs syncKey="package-manager">
         <TabItem label="bun">
           ```bash
           bun add @tcswap/plugins
           ```
         </TabItem>
         <TabItem label="pnpm">
           ```bash
           pnpm add @tcswap/plugins
           ```
         </TabItem>
         <TabItem label="npm">
           ```bash
           npm install @tcswap/plugins
           ```
         </TabItem>
         <TabItem label="yarn">
           ```bash
           yarn add @tcswap/plugins
           ```
         </TabItem>
       </Tabs>

       Install for interactions with specific protocols or products.
       Install the main plugins package below; you can then import specific plugin integrations as needed (e.g., `ThorchainPlugin` from `@tcswap/sdk`).
       By building your own plugins, you can extend USwap to support new protocols, products or actions.
       Currently USwap includes plugins for Chainflip, Thorchain, EVM, Radix, and more.
     </TabItem>

     <TabItem label="Toolboxes">
       Toolboxes provide foundational, chain-specific functionalities used internally by wallets and plugins.
       They are included when you install `@tcswap/core` and typically **don't need separate installation** unless you are building custom integrations or need direct access to low-level chain functions.
       Check out [Toolbox Usage](/start/toolbox-usage) for more information on advanced scenarios.
     </TabItem>
   </Tabs>

3. **Configure USwap**

   ```typescript
   import { USwap, Chain, ThorchainPlugin, EVMPlugin, keystoreWallet, ledgerWallet } from '@tcswap/sdk';

   const uSwap = USwap({
     config: {
       apiKeys: { uSwap: 'your-uswap-api-key' }
     },
     plugins: { ...ThorchainPlugin, ...EVMPlugin },
     wallets: { ...keystoreWallet, ...ledgerWallet },
   });


   await uSwap.connectKeystore([Chain.Ethereum], 'your mnemonic phrase here');
   const wallet = uSwap.getWallet(Chain.Ethereum);
   ```
</Steps>

<Aside type="tip">
  With the Core implementation, you only include the dependencies you actually
  need, resulting in a smaller bundle size.
</Aside>

## Common Use Cases

No matter which implementation you choose, you can use these common patterns:

<CardGrid>
  <Card title="Check Balances" icon="seti:database">
    ```typescript

    import { USwap, Chain } from '@tcswap/sdk';

    const uSwap = USwap({
      config: { apiKeys: { uSwap: 'your-uswap-api-key' } },
    });

    const balance = await uSwap.getBalance(Chain.Bitcoin, true);


    const allWallets = uSwap.getAllWallets();
    ```

  </Card>

  <Card title="Transfer Assets" icon="seti:send">
    ```typescript

    import { AssetValue, Chain, USwap, keystoreWallet } from '@tcswap/sdk';


    const uSwap = USwap({
      config: { apiKeys: { uSwap: 'your-uswap-api-key' }, wallets: { ...keystoreWallet } },
    });

    await uSwap.connectKeystore([Chain.Ethereum], 'your mnemonic phrase here');


    const assetValue = AssetValue.from({
      chain: Chain.Ethereum,
      symbol: 'ETH',
      decimal: 18,
      value: '0.1',
    });


    const txHash = await uSwap.transfer({
      assetValue,
      recipient: '0xRecipientAddressHere',
      memo: 'Optional memo',
    });
    ```

  </Card>

  <Card title="Perform Swaps" icon="seti:swap">
    ```typescript

    import { AssetValue, Chain, USwapApi, USwap, keystoreWallet } from '@tcswap/sdk';


    const uSwap = USwap({
      config: { apiKeys: { uSwap: 'your-uswap-api-key' }, wallets: { ...keystoreWallet } },
    });


    await uSwap.connectKeystore([Chain.Ethereum], 'your mnemonic phrase here');


    const quoteResponse = await USwapApi.getSwapQ({
      fromAsset: AssetValue.from({
        chain: Chain.Ethereum,
        symbol: 'ETH',
        value: '0.1'
      }),
      toAsset: AssetValue.from({
        chain: Chain.Bitcoin,
        symbol: 'BTC'
      }),
    });


    const swapTx = await uSwap.swap({ route: quoteResponse.routes[0] });
    ```

  </Card>
</CardGrid>

<Aside type="caution">
  Always secure sensitive information like mnemonic phrases. In production apps,
  never hardcode these values.
</Aside>
